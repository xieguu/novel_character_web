## ⚠️ 已知问题与弊端

### 1. LLM 配额限制 ⛔ **【紧急】**
**问题描述**：
- 人物提取功能依赖 LLM API
- 当账户配额耗尽时无法进行提取
- 错误信息：`LLM invoke failed: 412 Precondition Failed – your account has hit a usage exhausted`

**影响范围**：
- 用户无法上传文本进行人物提取
- 随机人物生成功能也会受影响

**解决方案**：
1. **短期**：升级 Manus 订阅或充值积分
2. **长期**：
   - 考虑集成本地 NLP 模型（如 jieba、HanLP）作为备选
   - 实现配额检查和友好的提示信息
   - 使用更便宜的 LLM API（如 Claude 的 batch API）

**优先级**：🔴 高

**优先级**：🟡 中

**当前状态**：

- ⚠️ 需要在生产环境中验证

**验证方法**：
1. 在生产环境中测试 PDF 导出功能
2. 检查生成的 PDF 质量和完整性

**优先级**：🟡 中

### 6. 搜索性能优化缺失 🔍
**问题描述**：
- 高级搜索功能已实现

- 大数据量下可能较慢

**优化方案**：
1. 添加数据库索引

**优先级**：🟢 低

## 📞 常见问题解决

### Q1: 人物提取失败怎么办？
**A**: 检查错误信息。如果是 LLM 配额问题，需要升级订阅。

### Q2: 如何添加新的人物字段？
**A**: 
1. 在 `drizzle/schema.ts` 中修改 `characters` 表
2. 运行 `pnpm drizzle-kit generate` 生成迁移
3. 在 `server/characterExtractor.ts` 中更新 LLM 提示
4. 在前端组件中添加对应的 UI

### Q3: 如何实现多用户实时协作？
**A**: 需要实现 WebSocket 服务器和冲突解决机制。参考 `collaborationService.ts` 中的权限检查逻辑。

### Q4: 如何优化搜索性能？
**A**: 添加数据库索引，实现缓存，考虑使用全文搜索引擎。
